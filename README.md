# Умный гардероб

Учебный прототип мобильного веб-приложения для школьников и студентов: утром подсказывает комплект одежды с учётом **погоды**, **расписания** (в т.ч. физкультура) и **чистых вещей** в гардеробе.

## Технологии

| Слой        | Стек                          |
|------------|--------------------------------|
| Клиент     | HTML, CSS, JavaScript (без фреймворков) |
| Сервер     | Python 3, Flask (минимум зависимостей) |
| Данные     | SQLite (`wardrobe.db`)       |
| Погода     | имитация API (`POST /api/weather/refresh`) или запись в БД при первом запуске |

## Требования

- **Python 3.9+**
- **Flask**

## Установка и запуск

1. Клонируйте или скопируйте папку проекта на компьютер.
2. Откройте терминал в **корне проекта** (там, где лежит `app.py`).

Установка зависимости:

```bash
pip install flask
```

Запуск сервера:

```bash
python app.py
```

По умолчанию приложение доступно по адресу: **http://127.0.0.1:5000/**


После запуска откройте в браузере главную страницу. База **`wardrobe.db`** и папка **`uploads/`** создаются автоматически при первом обращении к серверу.

## Первый запуск и данные

При **пустой** таблице вещей сервер один раз заполняет демо-данные:

- несколько вещей разных типов и сезонов (часть **чистые**, одна — **грязная**, чтобы было видно фильтрацию);
- расписание уроков на **сегодняшнюю дату**;
- запись погоды на сегодня.

Погоду можно обновить «как из API» (без реальных HTTP-запросов к внешним сервисам): кнопка на главной или запрос `POST /api/weather/refresh` с телом `{"date": "YYYY-MM-DD"}` (дата опциональна).

## API (кратко)

Базовый URL совпадает с адресом сервера, пути начинаются с `/api/`.

- **Вещи:** `GET/POST /api/items`, `PATCH/DELETE /api/items/<id>`
- **Расписание:** `GET /api/schedule?date=...`, `PUT /api/schedule`
- **Погода:** `GET /api/weather`, `POST /api/weather/refresh`
- **Рекомендация:** `GET /api/recommend?date=...`
- **Надеть комплект:** `POST /api/outfits/wear` (тело с `top_id`, `bottom_id`, `shoes_id`, `outer_id`)
- **Постирать всё грязное:** `POST /api/items/wash_all`
- **Сохранённые комплекты:** `POST /api/outfits/save`, `GET /api/outfits/saved`

Подробнее смотрите комментарии в `app.py`.

## Учебный контекст

Проект рассчитан на **локальный** запуск без сложной инфраструктуры: один процесс Flask, файловая БД, минимум зависимостей. Режим `debug=True` в `app.py` удобен для разработки; для «показа кейса» достаточно запуска на своём ПК в той же сети не выставляйте сервер в интернет без понимания рисков.
